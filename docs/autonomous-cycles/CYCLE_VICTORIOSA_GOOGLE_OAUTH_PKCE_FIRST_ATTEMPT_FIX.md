# Victoriosa Google OAuth PKCE First-Attempt Fix

Date: 2026-06-07

## Incident

The first Google authentication attempt reached the callback without the PKCE
verifier and displayed the raw Supabase error. Repeating the login succeeded.

## Root Cause

OAuth initiation used a server route while callback completion occurred in the
browser. On Vercel Preview, using a configured site URL could also change the
hostname between initiation and callback. Both conditions break the
browser-bound PKCE verifier.

## Fix

- Start Google OAuth with the browser Supabase client.
- Build the callback from `window.location.origin`.
- Preserve `request.url` origin in compatibility and callback routes.
- Skip middleware session refresh on `/auth/callback`.
- Replace raw authentication errors with a generic user-facing message.
- Add regression coverage for browser initiation, same-origin callbacks and
  PKCE error redaction.
